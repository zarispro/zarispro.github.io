Windows Me

     Windows Me（Windows Millennium Edition）是16位/32位混合的Windows操作系统，由微软公司在2000年9月14日发布。 Windows Me同时也是最后一款基于MS-DOS的Windows 9X内核系列的Windows操作系统，正式版本号是：4.9.3000。其名字有三种意思：第一是纪念2000年，Me是英文中千禧年（Millennium）的意思；另外也指自己，Me在英文中是“我”的意思；此外Me还有多媒体应用的意义（多媒体英文为Multimedia）。
     Windows Me很可能是已经推出的Windows系列中广告宣传做的最少的一个。由于家庭用户对Windows 2000的价格和系统要求配置的不解，微软推出的Windows 9X生产线上的最后一个产品给我们带来了一线希望。尽管Windows Me还存在着一些自身的缺陷，但是它毕竟提供了比Windows 9X更良好的稳定性，并且还附带有不少改进和新技术，所以我们强烈推荐正在使用Windows 9x的用户升级到Windows Me。而对于那些匆忙升级到Windows 2000却又被其搞得晕头转向的用户，Windows Me则像是大夫给你开出的一剂良药，让你可以和使用Windows 98那样轻车熟路，可能你还会发现其实它在操作方面要更简易。
     2000年Windows Me终于最终完成并交付生产，而且微软确实在这一版本的操作系统中做了很大改动。由于Windows 2000开发小组要用一年的时间调整下一个消费型Windows，即以Windows 2000为基础的Windows NT 5.0，代号为“Whistler”（即Windows XP的前身）。Windows Me事实上成为该操作系统面世前的一个过渡，它将是面向未来的Windows所迈出的第一步。另一方面，Windows Me被它推出的时间所拖累，因为它的生命期可能只有12个月，所以微软并没有足够的时间去照顾稳定性等方面的问题。比如，Windows Me开发小组没有时间把Windows 2000中的透明图标、鼠标指针的特殊效果添加进去，而且它也不会像Windows 2000那样支持双CPU。
     但是Windows Me确实在某些方面做的很不错。它与Windows 2000相比在硬件和软件支持方面要略胜一畴，能够稳定地支持更多的硬件和软件，需要的系统资源也比Windows 2000要少得多，因此可以在配置较低的计算机上良好运行。在相同配置的电脑上，Windows Me比Windows 2000开关机的速度更快，从休眠状态的唤醒过程也比Windows 2000更加迅速。在单CPU的电脑上，Windows Me的反应也要快于Windows 2000。
     下面，我们就一起来看看为什么某些Windows用户有必要升级到Windows Me，而为什么有些用户还是最好等着Whistler（Windows XP）的推出。当然了，最后的决定权还是在于用户自己手中。但是Windows Me确实比Windows 2000适合的用户群更广泛，所以如果你不能肯定选择的哪一款操作系统，就请跟着我们一起往下看吧。首要任务：稳定性
     Windows Me在稳定性方面确实做了些引人注意的改进，这让那些Windows 95和Windows 98用户有足够的理由升级到Windows Me。首先，Windows Me已经摆脱了“实模式DOS”这一MS-DOS应用程序运行时所需的环境。这一改变将对系统有多方面的影响，当然大多数都是正面的、并且积极的，Windows Me的启动更快和稳定更强正是去掉实模式DOS的成果。不过它要用户付出的代价也是有的，一些古老的DOS应用程序在Windows Me下无法继续使用，让人奇怪的是还有一些比较老的Windows应用程序竟然也无法启动了。我们猜想其中的主要是因为Windows Me不再支持进入DOS模式，而至使那些程序无法正常运行。虽然去除了实模式DOS，但是经过我们测试古老的DOS游戏和应用程序大多数还是能在Windows Me下很好地工作的。另外，“MS-DOS”模式的选项也还保留，不过它要更为隐蔽一些。而像Partition Magic等一些需要重新启动进入DOS模式的应用程序如果想在Windows Me下继续施展威力的话，就需要软件制作者针对Windows Me进行版本的升级与更新。不过总的说来，其实大多数用户在日常操作中并不需要实模式DOS的环境，而且随着软件的发展那些需要实模式DOS的应用程序将越来越少，所以我们认为微软在Windows Me中去掉它是一个不错的决定。
     和Windows 2000一样，Windows Me为我们提供了系统文件保护功能(SFC)，这可以防止那些关键的系统文件被覆盖。SFP在后台悄悄工作，确保这些系统文件没有被删除或者覆盖，如果某一个安装程序用自己所带的文件覆盖了这些系统文件，SFP会自动恢复正确的文件。这对于提高系统的稳定性是个不坏的想法，而且SFP在Windows 2000中的表现也非常令人满意，因此微软将其加入到Windows Me中是值得称道的作法。
     “系统恢复”是Windows Me是独有的一项功能，它的主要目的是替换Windows 95/98中的文件备份功能，当然在Windows Me中同时也为那些认为备份功能不可缺少的用户保留了该项功能。系统恢复功能能够自动备份新装程序的主要文件，这样一旦出现问题用户就可以借此将系统恢复到程序安装前的运行状态。用户可以自己定制将系统恢复到什么时间以前的状态，不过，我们在试用中发现系统恢复功能虽然不错，但是它将占用相当大的硬盘空间，这甚至会令人产生微软究竟要通过Windows Me达到什么不可告人的企图的怀疑。总的来说，系统恢复功能并非简单地替换以前的备份功能，因为它并不是将系统中所有发生改变的文件都进行备份，而只是记录那些系统启动和运行所必须的文件，这个作法与许多专业的备份软件的设计相似，那么它是不是意味着又有一批软件将要下课。这要等用户都使用上Windows Me之后才好下定论。
     Windows Me中Windows更新项也增加了一个新功能：“自动更新”。Windows更新是在Windows 98中初次登台露面的，它要求用户自己打开微软的支持页面检查是否有更新内容。微软则仅是利用“重要更新通知”提示用户注意修正自己系统中存在的漏洞。而Windows Me的自动更新功能将会帮你省去不少麻烦，如果你同意启用这个功能的话，它将可以随时帮你搞定以前那些繁琐的操作。自动更新可以默默地注意着Windows更新页面上是不是有适合用户操作系统的新更新推出。一旦发现有更新推出，自动更新功能便可自动下载并安装。当然，如果你担心最新的更新文件过于庞大，你也可以不用这项功能。不过这实在是Windows的一项重大改变，如果有一天我们都能用上宽带就好了。
